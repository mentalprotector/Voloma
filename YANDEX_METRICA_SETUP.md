# Yandex Metrica Setup

## Что уже подготовлено на сайте

- Сайт использует production ID `108819606`.
- При необходимости ID можно переопределить через `NEXT_PUBLIC_YANDEX_METRICA_ID`.
- Скрипт Яндекс.Метрики не загружается сразу.
- Пользователь сначала видит баннер согласия на аналитику.
- Метрика и цели включаются только после нажатия `Разрешить`.
- Если пользователь выбирает `Только необходимые`, Метрика не грузится.

## Задание человеку с доступом к Яндекс.Метрике из РФ

1. Зайти в [Яндекс.Метрику](https://metrika.yandex.ru/).
2. Создать новый счётчик для сайта Voloma.
3. Указать фактический домен сайта.
4. Включить нужные опции счётчика:
   - `Вебвизор` - по решению владельца сайта.
   - `Карта кликов` - можно включить.
   - `Отслеживание ссылок` - можно включить.
5. После создания счётчика передать только числовой ID.

## Что сделать после получения ID

Счётчик уже создан: `108819606`.

Для обычного деплоя отдельная переменная окружения не обязательна: этот ID уже задан как production-default в коде. Если нужно заменить счётчик, на сервере можно добавить переменную окружения:

```env
NEXT_PUBLIC_YANDEX_METRICA_ID=12345678
```

После любого изменения ID нужно пересобрать и перезапустить сайт, потому что `NEXT_PUBLIC_*` переменные встраиваются в клиентский бандл при сборке.

## Проверка

   - до согласия в Network нет запросов к `mc.yandex.ru`;
   - после `Разрешить` появляется загрузка `https://mc.yandex.ru/metrika/tag.js`;
   - в Метрике появляются визиты.

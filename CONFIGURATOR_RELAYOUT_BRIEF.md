# Перевёрстка конфигуратора Voloma

## Зачем переделывать

Текущая версия стала компактнее, но архитектура экрана теперь конфликтует сама с собой:

- мобильный экран собран из трёх независимых блоков: галерея, белая карточка с параметрами и отдельный sticky CTA;
- десктоп и мобильный используют одну и ту же строковую схему параметров, хотя у них разная доступная ширина;
- блок параметров держится на горизонтальном скролле для большинства групп, из-за чего опции не помещаются и выглядят сломанно;
- низ экрана добивается искусственным spacer-блоком, что создаёт пустоту и визуально ломает фон;
- светлая белая карточка и белый CTA выбиваются из общей тёплой палитры страницы и особенно заметны рядом с галереей.

Вывод: конфигуратор лучше не латать точечно, а переверстать как единый responsive flow с разной композицией для mobile и desktop, но на общей логике выбора.

## Что сейчас ломает интерфейс

### Технические причины

- Страница дополнительно ограничивает контент по ширине до `31rem` на mobile/tablet: `src/app/configurator/page.module.css`.
- Сам конфигуратор снова ограничен `max-width: 31rem`: `src/components/configurator/configurator.module.css`.
- На mobile controls-панель выезжает поверх медиа через `margin-top: -16px`, а затем компенсируется отдельным sticky CTA и spacer-блоком.
- Каждая строка параметров построена как `label + horizontal scroller`, поэтому длинные подписи и pill-кнопки не имеют естественного переноса.
- Для mobile CTA используется отдельный компонент и отдельный визуальный контейнер, который не является частью того же surface, что и параметры.
- Галерея на mobile имеет фиксированную высоту от viewport (`44vw`, `max-height: 200px`), поэтому верх кажется слишком низким, а белая часть визуально доминирует.

### Симптомы в UI

- Форма и тип дерева не помещаются по ширине и выглядят обрезанными.
- Под опциями образуется большое пустое белое поле.
- CTA визуально оторван от блока выбора.
- Цвет surface у галереи, controls и CTA не согласован.
- На desktop правая колонка слишком узкая для контента при сохранении больших pill-элементов.

## Целевое решение

### 1. Пересобрать layout экрана

Mobile:

- один вертикальный поток без наложения карточек;
- шапка, медиа, параметры, summary, CTA идут последовательно;
- sticky CTA остаётся только если действительно даёт пользу и не дублирует основной CTA.

Desktop:

- двухколоночный layout;
- слева галерея;
- справа единый блок конфигурации;
- summary + CTA закрепляются внизу правой колонки, но остаются частью той же панели.

### 2. Убрать горизонтальный скролл из основных групп

Для групп `Форма`, `Размер`, `Тип дерева`:

- перейти с `flex + overflow-x` на `grid` или `flex-wrap`;
- разрешить перенос опций на следующую строку;
- выровнять минимальную высоту контролов и внутренние отступы;
- на mobile label можно переносить над options, а не держать слева.

Для группы `Цвет`:

- оставить компактный ряд swatch-элементов;
- подписи либо уменьшить, либо перенести в подпись выбранного значения;
- активное состояние сделать чище, без лишнего визуального шума.

### 3. Развести mobile и desktop композицию строк параметров

Mobile row:

- вертикальная схема: label сверху, control group снизу;
- больше воздуха между группами;
- не пытаться экономить высоту ценой ломки ширины.

Desktop row:

- двухколоночная схема: компактный label слева, controls справа;
- можно сохранить более плотную структуру.

### 4. Собрать единый surface

- отказаться от отдельной белой карточки, которая налезает на галерею;
- привести gallery area, controls panel и CTA к общей палитре surface;
- использовать один фон панели и один тип разделителей;
- пустой spacer-блок удалить, высоту формировать нормальным контентным ритмом.

### 5. Пересобрать нижнюю часть

Нужно выбрать один из двух вариантов:

Вариант A, предпочтительный:

- summary и CTA встроены в конец панели параметров;
- на mobile дополнительно появляется компактная sticky-строка только с ценой и кнопкой, когда основной CTA ушёл выше viewport.

Вариант B:

- sticky CTA полностью убрать;
- внизу оставить обычный summary block + CTA как часть страницы.

Рекомендация: начать с варианта B, потому что он проще, чище и снимет часть конфликтов по высоте. Sticky-CTA вернуть только если после теста реально просядет конверсия или потеряется доступность действия.

## Предлагаемая структура компонентов

- `ConfiguratorLayout`
  - отвечает только за mobile/desktop композицию;
- `OptionGroup`
  - универсальная группа параметра;
- `SegmentedOptions`
  - форма, размер, тип дерева;
- `ColorOptions`
  - swatches;
- `SummaryCard`
  - название, конфигурация, срок, цена;
- `PrimaryCTA`
  - кнопка Telegram;
- `StickyCTA`
  - опциональный mobile-only слой, если останется нужен.

Важно: логика выбора и расчёта цены остаётся в `Configurator.tsx` или выносится в hook, а layout/components становятся максимально презентационными.

## Объём работ

### Этап 1. Layout

- убрать отрицательное наложение controls на gallery;
- убрать `mobileBottomSpacer`;
- собрать единый порядок секций для mobile;
- нормализовать desktop grid и ширину правой колонки.

### Этап 2. Option rows

- ввести отдельные mobile и desktop стили групп;
- заменить horizontal scroll на wrap/grid;
- поправить размеры pill-кнопок, типографику и интервалы.

### Этап 3. Summary/CTA

- объединить summary и CTA с основной панелью;
- убрать дублирование visual surface;
- решить судьбу sticky mobile CTA после теста.

### Этап 4. Visual polish

- выровнять фоновые поверхности;
- уменьшить ощущение пустоты;
- проверить hover/focus/pressed states;
- проверить safe-area отступы на iPhone.

## Критерии приёмки

- На ширинах `320`, `360`, `375`, `390`, `414` ни одна группа опций не требует горизонтального скролла.
- На mobile нет визуально случайного пустого белого блока после опций.
- CTA не дублируется как два равноправных больших блока на одном экране.
- На desktop при ширинах `1024`, `1280`, `1440` правая колонка не выглядит пережатой.
- Фон и surface конфигуратора визуально согласованы с общей палитрой Voloma.
- Все controls доступны с клавиатуры и имеют заметный `focus-visible`.
- Цена и выбранная конфигурация всегда остаются читаемыми без наложений и обрезания.

## Практическое решение по реализации

Рекомендую делать не "косметический фикс", а такую последовательность:

1. Сначала пересобрать DOM-структуру mobile/desktop.
2. Потом заменить строки параметров на wrap/grid.
3. Потом принять решение по sticky CTA.
4. После этого уже шлифовать размеры, цвета и ритм.

Если начать с точечных CSS-фиксов поверх текущей схемы, высокая вероятность, что мы просто замаскируем проблему и сохраним хрупкий layout.
